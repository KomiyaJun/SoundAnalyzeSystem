# 作品タイトル 概要
**タイトル:**
SoundAnalyzeSystemDemo

**概要:**
パート別(ドラムやベースなど)再生を行える機能などを持ったSoundManagerを利用し、BGMのリアルタイムFFT解析を通じて、音と連動変形するオブジェクトやテンポ検知システムなどを構築したプロジェクトです。

# 主な機能と特徴
***プロジェクトの主な機能と特徴*** 
本プロジェクトでは、楽曲（BGM）を単に再生するだけでなく、オーディオデータをリアルタイムに解析し、ゲーム内の通知システムやオブジェクトの視覚的変化と高度に同期させるオーディオ・リアクティブ・システムを構築しています。

***1. レイヤー別・インタラクティブなサウンド管理 (Sound フォルダ)***
再利用可能なサウンドマネージャー: 

**UnitySoundManager.cs**
 を中心に実装し、複数音源の再生・停止・ボリューム管理を安全に行う仕組みを構築しています。
パート（レイヤー）別の動的制御: BGMを単一のファイルとして扱うのではなく、パート（ドラム、ベース、メロディ等）ごとに分割して同期再生しています。さらに、状況に応じて特定のパートのみを有効・無効化（TogglePart / AddPart / RemovePart）できるインタラクティブなサウンドシステムを実現しています。

**2. FFTを用いたリアルタイムオーディオ解析 (SoundAnalyzer フォルダ)**
高速フーリエ変換（FFT）の活用: 

**AudioAnalyzer.cs**
**AudioMixerAnalyzer.cs**
 により、再生中の音声からサンプリングした波形データをFFT解析し、周波数スペクトル（SpectrumData）をリアルタイムに抽出しています。
帯域（バンド）別の抽出処理: 単なる全体の音量ではなく、指定した周波数帯の平均音量を取得する機能（GetBandAverage(min, max)）を実装。これにより、「低音域（キック・ベース）」や「高音域（シンバル）」といった特定サウンドに絞った緻密な値が取得可能になっています。

***3. 音声解析データを活用したゲームギミック・視覚表現の同期***
解析された周波数成分やビートのデータを他クラス（UIやオブジェクト）に渡し、音楽と完全にリンクした表現を実現しています。

BPM・テンポ検知による通知システム: 

**BpmAnalyzer.cs**
を用いて、特定の周波数のピークを検知。楽曲のビートやテンポのタイミングに合わせて、ゲーム内の通知イベント等を正確に発火させます。
オーディオ・リアクティブなオブジェクト変化: 

**AmbientParticleSync.cs**
や 
**Visualizer.cs**
などを通じ、取得した低周波・高周波のデータに基づいてパーティクルの生成量、オブジェクトのトランスフォーム（拡大縮小）、発光（BeatLightEffect.cs）を連動させています。

# 担当箇所、注力ポイント
**・担当箇所**
個人開発なのでプログラミングについては基本自分の記述したものになっています。
一部、UniTaskやDOTweenを利用しています。

**・注力ポイント**
**1.密結合を避けるアーキテクチャ設計(サービスロケータ&イベント駆動)**
**Service Locatorパターンの導入:**
シングルトンの乱用によるクラス間の密結合とテスト困難化を防ぐため、サービスロケータパターンを導入
依存関係を明確にし、拡張性と堅牢な状態管理を実現しました。
**Event * Actionによる疎結合:**
FFT解析の結果(ビート検知など)を直接他クラスの関数で呼ぶのではなく、UnityEvent / C# Action を介したイベント(オブザーバーパターン)にすることで、
UIやエフェクト側が解析側の実装を知らなくても動作する疎結合な設計にしました。

**2.CPU負荷からGPU負荷へのオフロード(パフォーマンスチューニング)**
**シェーダーを活用したパフォーマンス負荷の劇的な改善:**
スペクトログラムのUI表示について、当初はCPU側で毎フレーム全ピクセルのテクスチャ切り替え(Texture2D.Seto\Pixel等)を行っていましたが、激しい処理落ちが発生。
これを解決するためにCPU処理を最小限の「最新1列の書き込み」に留め、過去の履歴のスクロール移動は**GPU(シェーダーのUVスクロール)に処理を委譲することで、描画負荷を大幅に削減しました。

**3.ヒステリシスを利用した高精度なビート検知**
**ノイズに強いヒステリシス特性の実装:**
ビート検知ロジックにおいて、単一の閾値(感知ライン)では音波の微小なブレで何度も通知が発生する「チャタリング」が発生しました。
これに対し、ONになる閾値とリセットされるための下限閾値の2つを設ける状態遷移を実装
これにより、バスドラムなどの特定音を極めて正確に、かつノイズに騙されずに安定して取得できるようアルゴリズムを工夫しました。

**4.特定周波数帯の柔軟な抽出機能**
「低音域(バスドラムなど)や「高音域(シンバルなど)」、ターゲットにしたい音源に的を絞れるよう、FFTの配列から任意の周波数平均値を計算して切り出せる汎用的なAPIを構築しました。

## 操作方法
**S,W** : 左右移動

**Space** : ジャンプ

**Q,E** : BGMの再生スピード変更

**F** : インタラクト

インタラクト可能な物は2つあり、「説明バナー」と「アナライズカスタマイザー」の２つです。
説明バナーはインタラクトオブジェクト付近のギミックについて解説しています。

**LeftArrow,RightArrow** または**右下、左下のボタン** : ページの移動

アナライズカスタマイザーはFFTによる解析の対象の切り替え(各パートのオン/オフ)と各パートの音量の変更が可能です。

**ボタン(Drumなど)** : ボタンに記述されているパートの解析のオンオフを切り替えます。

**スライダー** : 曲の音量自体を変更します。 解析結果にも影響が出ます。
